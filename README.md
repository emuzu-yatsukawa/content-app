# 完全ローカル完結型 タスク計測・実績可視化システム 技術設計書

## 1. 文書概要
本書は、実装済みのシングルページWebアプリケーションについて、現行コードの仕様に基づき整理した技術設計書である。対象システムは、タスク計測、作業ログ蓄積、経験値管理、日次・週次の可視化、AI風の日報生成体験を、すべてブラウザ内で完結させる。

ブラインド審査に対応するため、本書には作成者を特定できる個人情報を含めない。

## 2. システムコンセプト
本システムの中核は、以下の4点である。

1. タイマー計測を起点に、作業ログを自動で記録できること。
2. LocalStorageをフル活用し、獲得EXP・グラフ・カレンダーのドット・AI日報をリアルタイムに連動させること。
3. 外部APIを一切使用せず、情報漏洩リスクを100%排除した完全ローカル完結型のAI疑似体験を提供すること。
4. 1.5秒の「AI分析中...」ローディング演出により、単なる帳票出力ではないリッチなUXを実現すること。

## 3. 審査向け訴求ポイント

### 3.1 タイマー機能と作業ログ記録
- タスク名を入力して計測開始すると、秒単位でタイマーが進行する。
- 計測終了時に、タスク名・開始時刻・終了時刻・作業秒数・獲得EXPを1件のログとして記録する。
- 作業時間に応じてEXPを自動付与し、レベル表示と進捗バーへ即時反映する。
- 選択日の実績一覧へその場で反映されるため、記録から振り返りまでが1動作で完結する。

### 3.2 LocalStorage連動のリアルタイム統合設計
- 作業ログ本体はLocalStorage内のアプリ状態として永続化される。
- 保存済みログを基に、以下の表示がリアルタイムで同期更新される。
  - 経験値表示とレベル表示
  - 日別ドーナツグラフ
  - 週間棒グラフ
  - カレンダー上の実績ドット
  - 日次AI日報パネル
- ゴミ箱ボタンでログを削除すると、ログ一覧だけでなくEXP、グラフ、カレンダー表示まで即時再計算される。
- AI日報は日付単位で別管理され、生成・削除の結果がLocalStorageへ即時保存される。

### 3.3 完全ローカル完結型AI疑似体験
- AI日報は外部API通信ではなく、当日ログの集計結果からブラウザ内ロジックで生成する。
- 生成内容は「業務サマリー」「生産性分析」「翌日へのアドバイス」の3区分で構成される。
- 通信先サーバー、推論API、クラウド保存を前提としないため、業務ログが端末外へ送信されない。
- これにより、機密情報を含み得る日次業務データに対しても、高い説明性と安全性を両立できる。

### 3.4 1.5秒ローディング演出によるUX強化
- AI日報生成時には、即時出力ではなく1.5秒の待機演出を挿入する。
- 画面上には「AI分析中...」のアニメーション付きローディング表示を出し、処理体験に没入感を与える。
- 完全ローカル処理でありながら、利用者にはAIが分析しているような自然な操作感を提供する。

## 4. システム概要

### 4.1 実装方式
- 形式: シングルページアプリケーション
- 実装: HTML / CSS / Vanilla JavaScript
- UI補助: Tailwind CSS CDN
- グラフ描画: Chart.js
- データ保存: ブラウザLocalStorage
- サーバー: 不要

### 4.2 画面構成
画面は大きく以下の3領域で構成される。

1. 左ペイン
   - 月間カレンダー
   - 日別 / 週間のグラフ切替表示
2. 右上ペイン
   - タスク名入力
   - タイマー表示
   - 計測開始 / 計測終了ボタン
3. 右中段から下段
   - 選択日の作業実績テーブル
   - AIワンボタン日報表示領域

ヘッダーにはレベル・EXPバーを常時表示し、利用継続の動機付けを行う。

## 5. 機能設計

### 5.1 タスク計測機能
- 利用者は任意のタスク名を入力して計測を開始できる。
- 計測中は入力欄をロックし、誤操作を防止する。
- 終了時に計測秒数を確定し、ログを自動生成する。
- タスク名が空の場合は、表示上「名無しのタスク」として扱う。

### 5.2 作業ログ管理機能
- ログは日付ごとに参照され、選択日の実績として一覧表示される。
- 一覧には、タスク名・完了時刻・作業時間・獲得EXP・削除操作を表示する。
- 表示件数は新しいものから最大5件である。
- ログ削除時は対象レコードを除去し、付与済みEXPも減算する。

### 5.3 経験値・レベル管理機能
- 獲得EXPは作業秒数と同値で算出する。
- 100EXPを1レベルの閾値とし、到達時には自動でレベルアップする。
- レベルアップ時には通知を表示し、ヘッダーのレベル値と進捗バーを更新する。
- ログ削除時には累積値から逆算し、レベルと現在EXPを整合的に再計算する。

### 5.4 カレンダー可視化機能
- 月間カレンダーを画面上で生成し、前月・翌月へ遷移できる。
- 選択日は強調表示し、本日は別スタイルで識別する。
- ログが存在する日付にはドットを表示し、記録日を一目で把握できる。
- 日付選択に応じて、実績一覧・グラフ・AI日報が同時に切り替わる。

### 5.5 グラフ可視化機能
- 本日タブでは、選択日のタスク別作業分布をドーナツグラフで表示する。
- 週間タブでは、選択日を終点とした過去7日間の作業時間推移を棒グラフで表示する。
- 日別ログが存在しない場合は、グラフの代わりに空状態メッセージを表示する。
- グラフは1つのキャンバスを共有し、表示切替時に再描画する。

### 5.6 AIワンボタン日報機能
- 選択日のログが1件以上ある場合のみ、日報生成を実行できる。
- 生成される日報は以下の3要素で構成する。
  - 業務サマリー
  - 生産性分析
  - 翌日へのアドバイス
- 日報は日付キー単位で保存され、同じ日を再表示した際に再利用される。
- 日報削除ボタンにより、当該日の保存済み日報のみを即時削除できる。

## 6. データ設計

### 6.1 永続化キー
- アプリ状態: ai_comp_app_state
- AI日報: aiReports

### 6.2 アプリ状態
アプリ状態は以下の要素を保持する。

| 項目 | 内容 |
| --- | --- |
| level | 現在レベル |
| exp | 現在レベル内のEXP |
| taskLogs | 作業ログ配列 |

### 6.3 作業ログデータ
各ログは以下の情報を持つ。

| 項目 | 内容 |
| --- | --- |
| id | 一意識別子 |
| taskName | タスク名 |
| startTime | 開始日時 |
| endTime | 終了日時 |
| durationSeconds | 作業秒数 |
| expGained | 獲得EXP |

### 6.4 AI日報データ
AI日報は日付文字列をキーとする連想配列で管理する。

| 項目 | 内容 |
| --- | --- |
| summary | 業務サマリー |
| analysis | 生産性分析 |
| advice | 翌日へのアドバイス |
| generatedAt | 生成日時 |

## 7. 処理フロー

### 7.1 タスク記録フロー
1. タスク名を入力する。
2. 計測開始を押下する。
3. タイマーが1秒ごとに更新される。
4. 計測終了を押下する。
5. ログを生成してLocalStorageへ保存する。
6. 実績一覧、EXP表示、カレンダー、グラフを再描画する。

### 7.2 ログ削除フロー
1. 実績一覧のゴミ箱ボタンを押下する。
2. 対象ログを配列から削除する。
3. 対応するEXPを減算する。
4. LocalStorageへ再保存する。
5. 実績一覧、ヘッダー、カレンダー、グラフを即時更新する。

### 7.3 AI日報生成フロー
1. 選択日のログ件数を確認する。
2. ログが存在する場合、生成ボタンを一時無効化する。
3. 「AI分析中...」ローディング演出を1.5秒表示する。
4. ログ集計結果から日報文面をローカル生成する。
5. 日付キー単位で保存し、表示を更新する。

## 8. 完全ローカル設計の安全性
- 作業ログ、経験値、日報データはすべて利用者のブラウザ内に保存される。
- 外部APIへの送信処理、認証連携、サーバー保存処理は実装していない。
- そのため、入力データが外部サービスへ送信される経路を持たない。
- 業務ログを扱うアプリケーションとして、説明責任の明確な安全設計である。

## 9. UX設計上の工夫
- タイマー停止直後に実績が反映されるため、入力完了の手応えが明確である。
- カレンダー、グラフ、EXPが同時に変化することで、記録行為の価値を視覚的に理解しやすい。
- AI日報の生成演出により、分析結果を見る体験そのものに期待感を持たせている。
- すべて1画面で完結するため、画面遷移なしで日次記録から振り返りまで実施できる。

## 10. 現行実装の到達点
本システムは、単なるタイマーやメモツールではなく、以下を統合した実装済みプロダクトである。

1. タイマー計測による正確な作業ログ収集
2. EXPとレベルによる継続利用設計
3. カレンダーとグラフによる日次・週次の可視化
4. LocalStorageを核とした完全同期型データ設計
5. 完全ローカル完結型のAI疑似体験

特に、ログ追加とログ削除の双方で表示系データが破綻なく連動する点は、実利用を意識した完成度の高い実装ポイントである。
