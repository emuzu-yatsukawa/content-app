# タスク計測・実績可視化システム 技術設計書（Supabase Cloud版）

## 1. 文書概要
本書は、実装済みのシングルページWebアプリケーションについて、現行コードの仕様に基づき整理した技術設計書である。対象システムは、ユーザー認証、タスク計測、作業ログ蓄積、経験値管理、日次・週次の可視化をSupabaseを通じて実現し、マルチユーザー対応のクラウド型システムとして構成されている。

ブラインド審査に対応するため、本書には作成者を特定できる個人情報を含めない。

## 2. システムコンセプト
本システムの中核は、以下の4点である。

1. タイマー計測を起点に、作業ログを自動で記録し、Supabaseクラウドへ永続化できること。
2. ユーザー認証により、個々のユーザーが独立したデータセットで管理でき、マルチユーザー対応を実現すること。
3. Supabase連携により、タスクログ、レベル、EXPをリアルタイムに同期し、複数デバイス間での一貫性を保証すること。
4. 画面上には「☁️ クラウド同期済」等の同期状況インジケーターを表示し、ユーザーに対してクラウド同期の状態を可視化するリッチなUXを実現すること。

## 3. 審査向け訴求ポイント

### 3.1 タイマー機能と作業ログ記録
- タスク名を入力して計測開始すると、秒単位でタイマーが進行する。
- 計測終了時に、タスク名・開始時刻・終了時刻・作業秒数・獲得EXPを1件のログとして記録する。
- 作業時間に応じてEXPを自動付与し、レベル表示と進捗バーへ即時反映する。
- 選択日の実績一覧へその場で反映されるため、記録から振り返りまでが1動作で完結する。

### 3.2 ユーザー認証とマルチユーザー対応設計
- Supabase Authentication により、メールアドレスとパスワードでのセキュアなユーザー登録・ログインを実現する。
- ユーザーごとに独立した user_profiles テーブルレコードを管理し、level・exp を個別に保持する。
- ユーザーごとに独立した task_logs テーブルレコードを管理し、作業実績を個別に蓄積する。
- 各ユーザーの作業ログは user_id キーで関連付けられ、ログアウト時には画面がリセットされ、他ユーザーのデータへのアクセスを防止する。

### 3.3 Supabase連携によるリアルタイム同期設計
- 作業ログ本体及びユーザーレベル・EXPはSupabase内の対応テーブルへ永続化される。
- Supabase から保存済みデータを基に、以下の表示がリアルタイムで同期更新される。
  - 経験値表示とレベル表示
  - 日別ドーナツグラフ
  - 週間棒グラフ
  - カレンダー上の実績ドット
- ゴミ箱ボタンでログを削除すると、Supabase から対象レコードが削除され、ログ一覧だけでなくEXP、グラフ、カレンダー表示まで即時再計算される。
- 同期状況は画面ヘッダーの「同期中...」「☁️ クラウド同期済」「❌ 同期エラー」インジケーターで可視化され、ユーザーに対してクラウドとの通信状態を直感的に伝える。

### 3.4 複数デバイス間での一貫性保証
- Supabaseをバックエンドとすることで、同一ユーザーが異なるデバイスからログインすると、最新のタスクログ・レベル・EXP が自動的に同期される。
- デバイスAでタスク記録を追加すると、デバイスBからアクセスした際に自動的に反映される。
- この設計により、ユーザーの業務データは端末非依存で一元管理される。

## 4. システム概要

### 4.1 実装方式
- 形式: シングルページアプリケーション
- 実装: HTML / CSS / Vanilla JavaScript
- UI補助: Tailwind CSS CDN
- グラフ描画: Chart.js
- バックエンド・認証: Supabase (PostgreSQL + Auth)
- データ保存: Supabase PostgreSQL テーブル（user_profiles, task_logs）
- 一時キャッシュ: ブラウザ LocalStorage（進行中のタイマー情報のみ保存）
- サーバー構築: 不要（Supabase SaaS を利用）

### 4.2 画面構成
画面は以下の要素で構成される。

**認証層（ログイン画面）:**
- ユーザー認証用のモーダルオーバーレイ
- メールアドレス、パスワード入力欄
- 「ログインする」ボタン
- 「新規アカウント登録」への切り替えボタン
- ログイン失敗時のエラーメッセージ

**メイン画面（認証後）:**
大きく以下の3領域で構成される。

1. ヘッダー領域
   - アプリケーションタイトル（「🚀 AIコンペ開発システム (Cloud)」）
   - 同期状況インジケーター（「接続中...」「☁️ クラウド同期済」「❌ 同期エラー」）
   - ログイン中のユーザーメール表示
   - ログアウトボタン
   - レベル・EXP進捗バー（常時表示）

2. 左ペイン
   - 月間カレンダー（前月・翌月遷移ボタン付き）
   - 日別 / 週間のグラフ切替表示

3. 右上ペイン
   - タスク名入力
   - タイマー表示
   - 計測開始 / 計測終了ボタン
   - 表示中の対象日付表示

4. 右中段から下段
   - 選択日の作業実績テーブル（タスク名、完了時刻、作業時間、獲得EXP、削除ボタン）
   - 削除ボタンなし

**フッター領域:**
- 著作権表記
- 「全データを初期化」ボタン（確認ダイアログ付き）

## 5. 機能設計

### 5.0 ユーザー認証機能
- Supabase Authentication による email/password ベースのユーザー管理を実装する。
- 初回アクセス時は認証モーダルが表示され、ログインまたは新規登録が強制される。
- 登録後、メールアドレスとパスワードでログイン可能になる。
- ログアウト時には全てのローカル状態がリセットされ、認証モーダルが再表示される。
- 認証セッションは Supabase が管理し、24時間の自動更新に対応する。

### 5.1 タスク計測機能
- 利用者は任意のタスク名を入力して計測を開始できる。
- 計測中は入力欄をロックし、誤操作を防止する。
- 計測中のタイマー情報（開始時刻、タスク名）は LocalStorage に一時保存され、ページリロード時の復帰に対応する。
- 終了時に計測秒数を確定し、ログを Supabase に永続化する。
- タスク名が空の場合は、表示上「名無しのタスク」として扱う。

### 5.2 作業ログ管理機能
- ログは Supabase task_logs テーブルに、user_id と共に永続化される。
- ログは日付ごとに参照され、選択日の実績として一覧表示される。
- 一覧には、タスク名・完了時刻・作業時間・獲得EXP・削除操作を表示する。
- 表示件数は新しいものから最大5件である。
- ログ削除時は Supabase から対象レコードを削除し、付与済みEXPも減算する。
- ログの作成・削除時には同期インジケーターが更新され、ユーザーにクラウド同期状態を通知する。

### 5.3 経験値・レベル管理機能
- 獲得EXPは作業秒数と同値で算出する。
- ユーザーのレベル・EXPは Supabase user_profiles テーブルに保存される。
- 100EXPを1レベルの閾値とし、到達時には自動でレベルアップする。
- レベルアップ時には通知を表示し、ヘッダーのレベル値と進捗バーを更新する。
- ログ削除時には累積値から逆算し、レベルと現在EXPを整合的に再計算する。
- レベル・EXP の変更は都度 Supabase へ同期される。

### 5.4 カレンダー可視化機能
- 月間カレンダーを画面上で生成し、前月・翌月へ遷移できる。
- 選択日は強調表示し、本日は別スタイルで識別する。
- ログが存在する日付にはドットを表示し、記録日を一目で把握できる。
- 日付選択に応じて、実績一覧・グラフが同時に切り替わる。

### 5.5 グラフ可視化機能
- 本日タブでは、選択日のタスク別作業分布をドーナツグラフで表示する。
- 週間タブでは、選択日を終点とした過去7日間の作業時間推移を棒グラフで表示する。
- 日別ログが存在しない場合は、グラフの代わりに空状態メッセージを表示する。
- グラフは1つのキャンバスを共有し、表示切替時に再描画される。

## 6. データ設計

### 6.1 Supabase テーブル構成
本システムは以下の2つの主要テーブルで構成される。

**user_profiles テーブル:**
| カラム | 型 | 説明 |
| --- | --- | --- |
| user_id | uuid (PK) | ユーザー固有ID（Supabase Auth から自動生成） |
| level | integer | 現在のレベル（デフォルト: 1） |
| exp | integer | 現在レベル内のEXP（0～100） |
| updated_at | timestamp | 最終更新時刻 |

**task_logs テーブル:**
| カラム | 型 | 説明 |
| --- | --- | --- |
| id | uuid (PK) | ログ固有ID |
| user_id | uuid (FK) | ユーザーID（user_profiles.user_id へのリンク） |
| task_name | text | タスク名 |
| start_time | timestamp | 開始日時 |
| end_time | timestamp | 終了日時 |
| duration_seconds | integer | 作業秒数 |
| exp_gained | integer | 獲得EXP（通常は duration_seconds と同値） |
| created_at | timestamp | レコード作成時刻 |

### 6.2 ブラウザ LocalStorage の用途
LocalStorage は以下の限定的な用途でのみ使用される。

| キー | 用途 |
| --- | --- |
| ai_comp_running_timer_start | 進行中タイマーの開始時刻（ページリロード後の復帰用） |
| ai_comp_running_timer_name | 進行中タイマーのタスク名 |

- タイマー終了後や全データ初期化時には自動的にクリアされる。
- ユーザーのタスクログ本体や状態情報は Supabase へ完全に委譲される。

## 7. 処理フロー

### 7.1 ユーザー認証フロー
1. 初回アクセス時に認証モーダルが表示される。
2. ユーザーがメールアドレスとパスワードを入力する。
3. 新規登録の場合、supabaseClient.auth.signUp() を呼び出す。
4. ログインの場合、supabaseClient.auth.signInWithPassword() を呼び出す。
5. Supabase から認証成功・失敗の応答を受け取る。
6. 成功時、handleAuthStateChange() がトリガーされ、モーダルが消える。
7. 成功時、loadStateFromSupabase() で user_profiles と task_logs を一括ロードする。

### 7.2 タスク記録フロー
1. ユーザーがタスク名を入力する。
2. 計測開始ボタンを押下する。
3. タイマーが秒単位で更新される（LocalStorage にタイマー開始時刻を保存）。
4. 計測終了ボタンを押下する。
5. 同期インジケーターが「⚡ 同期中...」に変わる。
6. supabaseClient.from('task_logs').insert() により Supabase に新規ログを保存する。
7. 同時に gainExp() で EXP を加算し、saveProfileToSupabase() でレベル・EXP を同期する。
8. 同期完了時、同期インジケーターが「☁️ クラウド同期済」に変わる。
9. renderLogs()、renderCalendar()、updateStatusUI() で画面全体を再描画する。

### 7.3 ログ削除フロー
1. 実績一覧のゴミ箱ボタン（🗑️）を押下する。
2. 同期インジケーターが「⚡ 同期中...」に変わる。
3. supabaseClient.from('task_logs').delete() により対象ログを Supabase から削除する。
4. deductExp() で対応するEXPを減算し、レベルを再計算する。
5. saveProfileToSupabase() でレベル・EXP を Supabase に同期する。
6. 同期完了時、同期インジケーターが「☁️ クラウド同期済」に変わる。
7. 画面全体を再描画する。

### 7.4 全データ初期化フロー
1. フッターの「全データを初期化」ボタンを押下する。
2. 確認ダイアログが表示される。
3. ユーザーが承認した場合：
   - supabaseClient.from('task_logs').delete().eq('user_id', userId) で全ログを削除
   - supabaseClient.from('user_profiles').upsert() で level=1, exp=0 に初期化
4. LocalStorage のタイマー情報をクリアする。
5. ブラウザ内のアプリ状態をリセットする。
6. 画面全体を初期表示に戻す。

## 8. クラウド同期設計における安全性とセキュリティ

### 8.1 ユーザーデータ隔離
- 各ユーザーのタスクログ・レベル・EXP は Supabase の user_id で厳密に隔離される。
- Supabase のセッション認証により、ログイン中のユーザーのみが自身のデータへアクセス可能である。
- ログアウト後は全てのローカルアプリ状態がクリアされ、他ユーザーのデータが画面に残らない。

### 8.2 クラウド保存のメリット
- ユーザーの業務ログは複数デバイス間で同期され、デバイス紛失時のデータ保護が実現される。
- Supabase のバックアップ・レプリケーション機能により、データの高可用性を実現する。
- ブラウザのキャッシュ削除やデバイス初期化後も、ログインにより全データが復帰する。

### 8.3 認証セッション管理
- Supabase Authentication により、平文パスワードはサーバーに送信されず、安全な方式で認証が行われる。
- セッショントークンは自動更新されており、24時間以上の連続利用に対応する。
- ログアウトボタン押下時に明示的にセッションが無効化される。

## 9. UX設計上の工夫
- タスク記録完了時に同期インジケーターが変化することで、クラウドへの保存が確認できる。
- タイマー停止直後に実績が反映されるため、入力完了の手応えが明確である。
- カレンダー、グラフ、EXP が同時に変化することで、記録行為の価値を視覚的に理解しやすい。
- ログイン・ログアウトにより個々のユーザーが自分のデータだけを閲覧できることで、プライベートな作業記録の安全性を確保している。
- すべて1画面で完結するため、画面遷移なしに日次記録から週次振り返りまで実施できる。

## 10. 現行実装（Supabase Cloud版）の到達点

本システムは、単なるタイマーやメモツールではなく、以下を統合した実装済みプロダクトである。

1. **ユーザー認証と多様なアクセス管理** - Supabase Authentication による email/password ベースのセキュアなユーザー登録・ログイン
2. **タイマー計測による正確な作業ログ収集** - 秒単位のタイマーと作業ログの自動生成
3. **EXPとレベルによる継続利用設計** - 獲得EXPと自動レベルアップによるゲーミフィケーション
4. **Supabase連携による永続化** - task_logs と user_profiles テーブルへのリアルタイム同期
5. **複数デバイス間でのデータ同期** - 同一ユーザーが異なるデバイスからアクセスした場合、最新のデータが自動的に反映
6. **カレンダーとグラフによる日次・週次の可視化** - 月間カレンダー、日別ドーナツグラフ、週間棒グラフ
7. **同期状況の可視化** - 「接続中...」「☁️ クラウド同期済」「❌ 同期エラー」インジケーターによるリアルタイムな状態通知

特に、ログ追加とログ削除の双方で Supabase との同期が正確に行われ、複数デバイス間でも表示系データが破綻なく連動する点は、実利用を意識した完成度の高い実装ポイントである。
