# コンセプトシート

## 1. 作品名

Task Quest Cloud

## 2. アクセスURL

https://venerable-cuchufli-e5ad43.netlify.app/

## 3. キャッチコピー

「記録すると育つ」認証付きタスク計測アプリ

## 4. コンセプト

### 4-1. 機能概要

タスク計測・実績可視化・EXP成長を1画面で回せるクラウド記録アプリ。

- 解決したかった課題:
作業記録は「入力が面倒」で継続しづらく、振り返り時も粒度が揃わない。そこで「計測開始・終了だけで記録できる」方式にした。
- このコンペでこのテーマを選んだ理由:
AI活用コンペの制作プロセス自体を計測対象にすると、開発時間の可視化と改善サイクルを回しやすい。
- 想定ユーザー:
日々の作業時間を簡単に残したい個人ユーザー、軽量な実績管理をしたいチーム内メンバー。

### 4-2. PRポイント

1画面に「入力・記録・可視化・進捗」を集約。認証、ログ永続化、EXP計算、日次/週次グラフ更新を連動させ、操作後に即フィードバックが返る体験にした。

### 4-3. AI活用のハイライト

- 効いた使い方:
UI文言の統一、状態遷移の整理、エラーハンドリング方針の明文化をAIに分担し、実装速度を上げた。
- ハマった使い方:
一度に広範囲を生成させると仕様の揺れが出やすく、機能単位で小さく指示した方が品質が安定した。

### 4-4. 意図的にやらなかったこと(既知の限界)

- 意図的にやらなかったこと:
カテゴリ管理、通知、複雑な分析画面は除外し、計測体験の速さを優先した。
- 既知の限界:
1. インフラ依存: Supabase接続不可時は同期できない
2. 分析粒度: 現状は日次/週次中心で、月次・比較分析は未対応
3. 運用面: 実運用にはRLS設定と監査ログ設計の強化が必要

### 4-5. 時間さえあればこんな拡張も視野に入れているという内容

- 時間があれば追加したい機能:
タグ集計、月次レポート、CSVエクスポート、週次レビュー自動コメント。


## 5. アクセス方法と動作確認方法(簡易版マニュアル)

- 動作環境: 最新版の Chrome / Edge / Firefox、インターネット接続（CDN・Supabase接続のため）
- ログイン: メールアドレス + パスワードで新規登録またはログイン
- テスト用既存アカウント: ID: test@company.com / パスワード: test1234

### 動作確認手順

1. 公開URL（https://venerable-cuchufli-e5ad43.netlify.app/）にブラウザでアクセス
2. ログイン画面で既存アカウント（test@company.com / test1234）または新規登録でログイン
3. タスク名を入力して「計測開始」→数秒後に「計測終了」
4. 実績テーブル、カレンダー、グラフ、EXP表示が更新されることを確認
